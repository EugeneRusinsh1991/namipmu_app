# Реализация системы визуального редактирования UI (UI Lab / Design System Editor)

Этот документ описывает создание мощного внутреннего инструмента (Visual Style Editor), который позволяет дизайнерам и разработчикам настраивать интерфейс в реальном времени, "на лету" меняя токены системы без прямого изменения исходного кода.

## 1. Архитектура "Design Tokens" (Основа)
Все визуальные параметры (цвета, отступы, типографика) должны быть абстрагированы от компонентов.
*   **За что отвечает:** Создает "единый источник истины" (Single Source of Truth).
*   **Детальная реализация:**
    *   **Уровень 1 (Global):** Базовые цвета (Palette), шкала отступов (Spacing Scale), сетка.
    *   **Уровень 2 (Semantic):** `primary-button-bg`, `card-border-radius`, `main-text-color`.
    *   **Хранение:** Использование CSS Custom Properties (Variables) в `:root`. Это позволяет менять стили без перерендера React-дерева, просто обращаясь к `document.documentElement.style.setProperty()`.
*   **Как выглядит:** Файл `tokens.json` или `theme.ts`, который компилируется в набор CSS переменных.

## 2. Изолированная среда (Sandbox / Canvas)
Создание отдельного окна или маршрута (например, `/ui-lab`), где компоненты рендерятся в чистом виде.
*   **За что отвечает:** Позволяет видеть компонент в отрыве от бизнес-логики, проверяя его поведение в разных состояниях.
*   **Детальная реализация:**
    *   **Iframe-песочница:** Использование iframe для полной изоляции стилей редактора от стилей самого приложения.
    *   **Стейты:** Отображение каждого компонента сразу в нескольких состояниях: `Normal`, `Hover`, `Focus`, `Active`, `Disabled`, `Error`.
*   **Вид:** Центральная область (Canvas) с сеткой, где можно переключать фон (светлый/темный/прозрачный) для проверки контрастности.

## 3. Интеллектуальный Редактор Свойств (Property Inspector)
Панель управления, которая автоматически подстраивается под выбранный элемент.
*   **За что отвечает:** Тонкая настройка каждого пикселя без написания кода.
*   **Детальная реализация:**
    *   **Типографика:** Полный контроль `font-family` (через Google Fonts API), `font-size`, `line-height`, `letter-spacing` (слайдеры с шагом 0.1).
    *   **Отступы (The Box Model):** Визуальный "квадрат" для настройки `padding` и `margin` отдельно для каждой стороны (Top, Right, Bottom, Left).
    *   **Border & Shadows:** Визуальный конструктор теней с возможностью наложения нескольких слоев (как в Figma).
    *   **Цвета:** Eye-dropper (пипетка) и поддержка прозрачности (RGBA/HSLA).
*   **Результат:** Правая панель с группами настроек, которые можно сворачивать (аккордеоны). Каждое изменение мгновенно обновляет переменную в памяти приложения.

## 4. Механизм "Live Sync" и Драг-н-дроп (Visual manipulation)
Возможность взаимодействовать с элементами прямо на холсте.
*   **За что отвечает:** Интуитивное изменение размеров и позиционирования.
*   **Детальная реализация:**
    *   **Resizers:** При наведении на компонент появляются "ручки" (handles), за которые можно тянуть, меняя ширину/высоту или `border-radius`.
    *   **Overlay Highlighter:** При наведении в редакторе на элемент, в превью подсвечивается его область (как в Chrome DevTools).
    *   **Context Menu:** Правой кнопкой мыши можно быстро скопировать стили конкретного элемента или сбросить их до дефолтных.

## 5. Мост между Редактором и Кодом (Persistence Layer)
Самый важный этап — превращение визуальных изменений в реальный код проекта.
*   **За что отвечает:** Сохранение результата работы дизайнера.
*   **Детальная реализация:**
    *   **Версионность:** Возможность сохранять "снимки" (Snapshots) настроек. Например, "Вариант 1 (Яркий)", "Вариант 2 (Минимализм)".
    *   **Smart Export:** Генерация не просто JSON, а готового TypeScript интерфейса или CSS-файла.
    *   **Websocket Server:** Маленький локальный сервер (Node.js), который слушает изменения в редакторе и автоматически перезаписывает файлы в папке `/src/styles/` вашего проекта. Это дает эффект "Hot Reload" для всей дизайн-системы.
*   **Вид:** Модальное окно с Diff-вьювером (сравнение: что было в коде vs что вы настроили в редакторе).

## 6. Визуализация результата (Layout приложения)
Интерфейс "Программы для настройки" должен быть максимально чистым:

1.  **Верхняя панель (Toolbar):**
    *   Выбор вьюпорта (Mobile, Tablet, Desktop).
    *   Переключатель тем (Светлая/Темная).
    *   Индикация связи с сервером (Live Sync status).
    *   Кнопка "Применить ко всему приложению".

2.  **Левая панель (Library):**
    *   Поиск по компонентам.
    *   Группировка: Atoms (Buttons, Icons), Molecules (Inputs, Cards), Organisms (Navigation, Modals).

3.  **Центральная зона (Stage):**
    *   Область, где живет выбранный компонент. Вокруг него отображаются линейки (Rulers) для замера расстояний.

4.  **Правая панель (Inspector):**
    *   **Вкладка "Styles":** Все CSS свойства.
    *   **Вкладка "Tokens":** Связь текущего элемента с глобальными токенами (например, "Этот цвет — это `brand-primary`?").
    *   **Вкладка "JSON":** Прямой доступ к коду текущего элемента для гиков.

## 7. Технические нюансы и безопасность
*   **Development Only:** Данный функционал должен подключаться только в `process.env.NODE_ENV === 'development'`. В продакшн сборку редактор попадать не должен.
*   **Performance:** Использование `React.memo` и оптимизированных слушателей для слайдеров, чтобы интерфейс не "лагал" при быстром изменении значений.
